# User Flow — Quiz Platform (Miro / FigJam)

> Диаграмма user flow для этапа проектирования. Эквивалент доски в Miro.

## Организатор

```
Landing → Регистрация (роль: организатор) → Dashboard
  → Создать квиз → Редактор вопросов → Сохранить
  → Запустить сессию → Лобби (код комнаты)
  → Старт квиза → Управление вопросами (Next / Reveal)
  → Финальный лидерборд → Профиль (история проведённых)
```

## Участник

```
Landing → Регистрация (роль: участник) → Ввести код комнаты
  → Лобби (ожидание) → Вопрос (таймер, ответ)
  → Результат вопроса → … → Финальный лидerboard
  → Профиль (история участия)
```

## Состояния live-сессии

```
LOBBY → QUESTION → REVEAL → QUESTION → … → FINISHED
```

- В `QUESTION` участники могут отправлять ответы
- В `REVEAL` показываются правильные ответы и начисленные баллы
- Сервер отклоняет ответы вне состояния `QUESTION` или после истечения таймера

## FigJam / Figma

После авторизации в Figma MCP создайте FigJam-диаграмму командой `generate_diagram` или вручную по схеме выше. Ссылку добавьте в `docs/design/FIGMA_LINK.md`.
